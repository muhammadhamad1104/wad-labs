Class Structure and Inheritance

The UML class diagram: centers on a User class with attributes like username, password, and role, serving as the base for specialized roles. Student and Staff inherit from User, ensuring shared properties while defining role-specific behaviors. Students gain methods like submitQuestion() and vote(), while Staff use answerQuestion() to provide solutions. This hierarchy promotes code reuse and clear separation of responsibilities.

Associations and Data Flow

The Question class acts as a hub, linked to a Student (submitter), Module (course context), and Answer (Staff response). Each Question aggregates Vote objects to track engagement and includes validation logic (checkProfanity()). The Module class ties questions to academic courses, while Vote and Answer classes ensure traceability with timestamps.

Key Points

  • Question associates with Student, Module, Answer, and Vote.
  • Vote tracks voter identity and timestamp for auditing.
  • Module groups questions by course (e.g., code, tutor).
